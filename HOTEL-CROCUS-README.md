# «Крокус» — відпочинковий комплекс у Татарові

[Українська](HOTEL-CROCUS-README.md) · [English](HOTEL-CROCUS-README.en.md)

Адаптивний односторінковий сайт для відпочинкового комплексу «Крокус» у Карпатах. Сайт знайомить гостей із номерами та послугами, показує розташування комплексу й приймає заявки на бронювання.

[Переглянути сайт](https://log1st1ck.github.io/Crocus-Tatariv/) · [Репозиторій проєкту](https://github.com/log1st1ck/Crocus-Tatariv)

![Головна сторінка сайту «Крокус»](assets/project-hotel-crocus-screenshot.png)

## Про проєкт

«Крокус» — статичний frontend-сайт без CMS і без збірника. Інтерфейс розроблено на чистих HTML, CSS та JavaScript, тому проєкт легко розгорнути на GitHub Pages або будь-якому іншому статичному хостингу.

Основна мета сайту — дати гостю всю потрібну інформацію в одному місці та скоротити шлях від знайомства з комплексом до заявки на бронювання.

## Що реалізовано

### Інформація про комплекс

- hero-блок із чіткими закликами до бронювання та перегляду номерів;
- опис комплексу, ключові переваги й коротка статистика;
- картки номерів із фото, місткістю, описом і ціною;
- окрема таблиця тарифів, включно з груповим розміщенням;
- каталог послуг: кухня, сауна, конференц-зал, активний відпочинок, паркінг, Wi-Fi, групові заїзди та побутові послуги;
- адреса, GPS-координати, маршрути автомобілем і громадським транспортом;
- інтерактивна карта Google Maps;
- контакти, правила поселення та виселення.

### Бронювання

- основна форма швидкого бронювання в контактному блоці;
- окреме модальне вікно бронювання, доступне з шапки та карток номерів;
- автоматичний вибір потрібного типу номера після натискання кнопки в його картці;
- вибір дат заїзду й виїзду через Flatpickr з українською локалізацією;
- заборона вибору минулих дат і контроль послідовності заїзду та виїзду;
- вибір кількості гостей у модальному вікні;
- санітизація та валідація імені, телефону, дат, типу номера й кількості гостей;
- зрозумілі повідомлення про помилки біля відповідних полів;
- надсилання заявки через Google Apps Script для подальшого запису в Google Sheets;
- модальне підтвердження після відправлення форми.

### Інтерфейс і взаємодії

- адаптивна верстка для desktop, tablet і mobile;
- мобільне меню з коректним керуванням станом та `aria-expanded`;
- галереї номерів і послуг із великим фото, мініатюрами, лічильником і навігаційними стрілками;
- керування галереями з клавіатури: `Escape`, `ArrowLeft` та `ArrowRight`;
- легкий паралакс hero-секції через `requestAnimationFrame`;
- підтримка системного налаштування `prefers-reduced-motion`;
- плавна навігація між секціями;
- fallback-зображення для галереї номерів.

### SEO

- семантична структура сторінки;
- оптимізовані `title`, `description`, `keywords` і geo-метатеги;
- Open Graph і Twitter Card для прев’ю посилань;
- JSON-LD розмітка типу `LodgingBusiness`;
- дані про адресу, координати, телефони, послуги та типи номерів у структурованій розмітці;
- змістовні `alt`-описи зображень;
- `canonical`, meta robots і локалізований український контент.

## Локальний запуск

Проєкт не потребує встановлення npm-пакетів або окремої збірки.

```bash
git clone https://github.com/log1st1ck/Crocus-Tatariv.git
cd Crocus-Tatariv
python3 -m http.server 8080
```

Після запуску відкрийте [http://localhost:8080](http://localhost:8080).

Інтернет потрібен для завантаження Google Fonts, Flatpickr, Google Maps і надсилання заявок через Google Apps Script.

## Налаштування перед публікацією

1. Вкажіть актуальну адресу Google Apps Script у константі `GOOGLE_SCRIPT_URL` у `script.js`.
2. Замініть `https://ВАШ-ДОМЕН.ua/` на реальний домен у `canonical`, Open Graph, Twitter Card та JSON-LD.
3. Додайте актуальне соціальне зображення `images/cover.jpg` або змініть шлях у метатегах.
4. Додайте `manifest.webmanifest` або приберіть його підключення з HTML.
5. Додайте фотографії `images/kitchen/1.jpg`–`8.jpg`, якщо галерея кухні має бути активною.
6. Перевірте кінцевий сценарій збереження заявки в Google Sheets після розгортання.

> Функції `sendTelegram()` і `sendTelegramModal()` мають історичні назви. У поточній реалізації заявки надсилаються через Google Apps Script; інтеграції з Telegram API немає.
